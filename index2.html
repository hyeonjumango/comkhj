주민과 주민 / 주민과 관리사무소가 원활한 소통을 하여 더 편리한 아파트 생활을 누릴 수 있게, 더 쉽게 아파트를 관리할 수 있는 편의 제공 서비스입니다.

주요 기능

  • 회원가입 및 로그인
  • 게시판 (공지사항, 건의합니다, 주민투표, 자유게시판, 주요 일정)
  • 차량관리 (입주차량, 방문차량)
  • 부대시설 예약 (독서실, 멀티코트장)
  • 관리자 (입주민 관리, 웹 사이트 관리)

담당 기능

  • 아파트 방문차량 예약 시스템 구현
  • 입주차량 관리 시스템 구현
  • 게시글 신고 기능
  • 이메일 인증
방문차량 메인

화면 설명
방문차량 기능의 메인페이지입니다. 입주민만 이용할 수 있으며 관리자로 로그인했을 시 접근 불가의 경고창이 뜨게 됩니다. '방문차량 예약' 버튼을 누르면 '방문차량 예약 페이지'로, '방문예약 목록' 버튼을 누르면 '방문차량 예약 내역 페이지'로 이동할 수 있습니다.

방문차량 예약

화면 설명
방문차량 예약 페이지에 들어가면 차량정보 입력칸이 표시됩니다. 방문날짜, 차량번호, 방문 목적, 비상 연락처를 입력하면 방문차량 내역이 등록됩니다. 등록한 내역은 방문차량 예약 내역 페이지에서 확인할 수 있습니다.
Input에 입력된 값들이 post 방식으로 Servlet으로 전달되면 session에 저장되어 있던 로그인 유저 정보와 함께 DB에 저장됩니다. 등록이 완료되면 '등록 내역 확인페이지'로 이동하게 됩니다. 방문일 input에 선택된 날짜와 현재 날짜를 비교하여 지난 날짜를 입력했을 경우 자바스크립트로 확인하여 경고창이 노출되며 등록이 진행되지 않습니다.

등록내역 확인

화면 설명
방문차량 등록이 완료되면 이동되는 '등록 내역 확인페이지'입니다. 등록한 내용을 확인할 수 있으며, '추가 등록하기' 버튼을 누르면 '방문차량 예약 페이지'로, '변경 및 삭제하기' 버튼을 누르면 '방문차량 내역 변경 및 삭제 페이지'로, '내역 보기' 버튼을 누르면 '방문차량 예약 내역 페이지'로 이동할 수 있습니다.

등록내역 변경 및 삭제

화면 설명
방문차량 등록 내역을 변경하거나 삭제할 수 있는 페이지입니다. Input에 변경할 내용을 입력하고 '변경하기' 버튼을 클릭하면 변경된 내용이 DB에 저장됩니다. '삭제하기' 버튼을 누르면 해당 내역은 더 이상 노출되지 않습니다. '목록으로' 버튼을 누르면 '방문차량 예약 내역 페이지'로 이동할 수 있습니다.

방문차량 예약 내역

화면 설명
방문차량 예약 내역을 확인할 수 있는 페이지입니다. Session에 저장되어 있는 접속한 유저의 정보를 읽어와 접속한 사용자 뿐만 아니라 같은 세대에 살고 있는 사용자가 등록한 내역을 모두 볼 수 있습니다. 등록한 내역은 방문현황이 '접수'라고 보이며 차량이 방문하여 관리자가 확인처리를 하였을 경우 '완료'라고 보이게 됩니다. 내역 수정은 접속한 사용자가 등록한 내역, 차량이 아직 방문하지 않았을 경우, 방문일이 지나지 않았을 경우만 가능하며, JSTL의 Core Tags를 사용하여 조회된 값을 판단하여 각 내역의 수정 버튼 활성화 여부가 결정됩니다. 이외의 경우에는 수정 불가 안내 모달창이 뜨게 됩니다. 방문일, 차량번호, 신청인에 따라 내역 검색도 가능합니다. 검색 input에 입력된 값에 따라 sql문을 다르게 적용하여 여러가지 조건의 검색이 가능하도록 구현하였습니다.

[관리자] 방문차량 메인

화면 설명
관리자가 아파트의 전체 방문차량 등록내역을 조회할 수 있습니다. JSTL의 Core Tags를 활용하여 DB의 값에 따라 내역의 ‘상태’ 정보가 다르게 표시됩니다. 접수 완료 시 ‘접수’, 방문 완료 시 ‘완료’로 표시됩니다. 예약일자, 입차현황, 차량번호, 방문세대에 따라 내역 검색도 가능합니다. 검색 input에 입력된 값에 따라 sql문을 다르게 적용하여 여러가지 조건의 검색이 가능하도록 구현하였습니다.

[관리자] 방문차량 확인

화면 설명
관리자는 차량이 아파트에 방문했을 시 차량정보를 등록 내역에서 확인하여 방문 처리를 완료할 수 있습니다. 내역의 상태를 클릭하게 되면 해당 내역의 접수 확인 모달창이 뜹니다. '확인하기' 버튼을 누르면 등록번호를 Servlet에 전달하여 DB의 차량 방문 상태에 해당하는 컬럼의 값을 '완료'로 변경시켜줍니다. '완료' 처리된 내역을 취소처리 할 수도 있습니다. 내역의 상태 변경은 당일에 해당하는 내역만 처리할 수 있습니다. 자바스크립트로 내역의 방문일과 방문처리일을 비교하여 다를 경우 alert창이 뜨며 더 이상 방문처리가 진행되지 않습니다. 검색 input에 입력된 값에 따라 sql문을 다르게 적용하여 여러가지 조건의 검색이 가능하도록 구현하였습니다.

[관리자] 방문차량 현장접수

화면 설명
등록내역이 없는 차량 방문시 현장접수가 가능하도록 구현하였습니다. '방문차량 현장접수' 버튼을 누르면 차량정보를 입력할 수 있는 모달창이 뜹니다. 당일에 해당하는 내역만 접수가 가능하며 현장접수로 등록된 내역은 신청인이 관리자로 표시되어 입주민의 예약 내역 페이지에서도 확인할 수 있습니다.
